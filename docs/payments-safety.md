# Payments safety

Patch 1 intentionally does not implement real money.

Use product language carefully:

- prefer "buy credits" over "deposit"
- prefer "earned credits" over "user balance available for withdrawal"
- keep purchased and earned credits separate

Before real payments:

- define terms
- define refund policy
- define dispute policy
- define payout rules
- add KYC/KYB requirements for earners if needed
- add fraud controls and limits
- add ledger reconciliation
- review Stripe/Stripe Connect design
- review compliance requirements by launch country
